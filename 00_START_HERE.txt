📡 API ENDPOINTS (7 Total)
================================================================================

Base URL: /api/friends/

GET     /                        Get all friends with location data
GET     /:friendId/location      Get specific friend's location
GET     /requests/pending        Get pending friend requests
POST    /request/send            Send friend request to user
POST    /request/accept          Accept pending request
POST    /request/reject          Reject/cancel request
POST    /:friendId/remove        Remove friend from list

All endpoints require JWT authentication.

🎓 HOW IT WORKS
================================================================================

User Adding a Friend:

1. User A clicks "➕ Add" button
2. Modal opens with search field
3. User A types friend's name
4. Results appear in modal
5. User A clicks "+ Add" button
6. Frontend: POST /api/friends/request/send
7. Backend: Create Friendship with status="pending"
8. Success message shown
9. User B sees request (when logs in)
10. User B can accept/reject request
11. On accept: Both added to each other's friendList

Viewing Friend Location:

1. User A checks checkbox next to Friend B
2. Frontend: GET /api/friends/:friendId/location
3. Backend: Validate friendship, return location
4. JavaScript: Create color-coded marker on map
5. Frontend: Subscribe to Socket.IO location updates
6. Every 5 seconds: Location data updates in real-time
7. Marker moves without page refresh
8. When User A unchecks: Marker removed, Socket.IO unsubscribed

Real-Time Updates:

1. Friend shares location every 5 seconds (automatic)
2. Browser Geolocation API gets coordinates
3. POST /api/location/update sends to backend
4. Backend saves to User model
5. Socket.IO emits 'receive-location'
6. Frontend updates selected friends' markers
7. Map updates in real-time
8. No page refresh needed
